서버로 파일을 전송하는 업로드라는 기술은 java 로 직접 구현할 경우, 개발자가 네트워크 통신,
	IO(입출력), FILE 등을 제어해야 한다., 따라서 실무에서는 이미 누군가가 제작해 놓은 라이브러리 이용
	[종류]
	유료 -
	무료 - Apache-FileUpload컴포넌트,
			Oreilly-Cos.jar (IT출판사에서Servlet책을 발간하며 만듬)[servlets.com]
	서버로 일반 텍스트가 아닌, 이미지, 동영상과 같은 파일(인간이 해석할수 없는 유형의 파일 : 바이너리 파일)
	반드시 POST 방식으로 전송해야 하며 특히나 incoding 타입도 multipart/form-data 라는 형식을 지정해야 한다.
	이때 multipart 의 의미는 서버로 전송하고자 하는 미디어 타입이 텍스트 뿐만 아니라 , 바이너리도 섞여 있는 복합유형을 의미
